# SanhuoAI 发布说明与下一阶段 Backlog（2026-02-26）

## 本轮发布说明

### 1) 写作工坊与改写链路

- 一键改写支持阈值参数（0-1）并持久化。
- 改写流程调整为“生成建议正文 -> 用户确认 -> 应用替换”，不再默认直改正文。
- 改写提示词强化节拍、背景资料、上下文约束。
- 逻辑冲突支持“单点建议正文”与“全章重写建议”两种策略，均需确认后应用。
- AI 痕迹检测改为弹窗展示；助手聊天支持清空。

### 2) 章节导航与界面一致性

- 章节快速跳转支持 `当前章/总章数` 交互（总章数动态）。
- 章节下拉与全局下拉统一为不透明样式，并支持滚动查看。
- 左侧导航支持收缩/展开并持久化状态。

### 3) 审核中心

- 新增“整本 / 指定章节”审阅范围切换。
- 评分卡做了归一化兜底，避免异常输出导致四项全 0。
- 问题列表与摘要展示改为结构化呈现，修复文本粘连。

### 4) 导入导出

- 新增项目导出：`JSON`、`TXT`、`Markdown`。
- 新增旧书导入：自动映射章节与主要板块数据。
- 导入后进行基础记忆回填，可直接进入续写流程。

### 5) 独立关系图谱

- 新增独立“关系图谱”板块（路由 + 侧栏入口）。
- 新增图谱查询 API（节点、边、证据片段）。
- 支持搜索聚焦、节点点击、关系详情显示。

## 验证结论

- 构建验证：`npm run build` 通过。
- 后端编译：`python -m py_compile ...` 通过。
- 回归脚本：
  - `agent/scripts/regression_memory_path.py` 通过。
  - `agent/scripts/regression_import_graph_smoke.py` 通过（含导入兜底与图谱性能烟测）。
- 路由冒烟：`/api/content/reviews*` 与 `/api/graph/relations` 已注册。
- 迁移安全：未检出破坏性 SQL 模式（增量迁移策略保持）。

## 已知风险与建议

- 当前自动化回归偏 smoke 粒度；建议后续补充前端 e2e（尤其是写作工坊交互链路）。
- 图谱性能目前是中等规模初测，未覆盖超大规模（>1k 节点）场景。

## 下一阶段 Backlog（建议优先级）

1. P1：写作工坊 e2e 自动化（章节跳转、改写确认应用、聊天清空、AI 痕迹弹窗）。
2. P1：导入器增强（更复杂章节标题样式、异常段落分块策略可配置）。
3. P1：图谱性能分层（边懒加载、采样渲染、超大图分区）。
4. P2：审核中心“问题一键生成修稿任务单”能力。
5. P2：导出模板增强（发布平台格式预设，如晋江/起点风格）。
6. P3：写作助手提示词可视化配置（首章规则、速度模式模板、约束开关）。
